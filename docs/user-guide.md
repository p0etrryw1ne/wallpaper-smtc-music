# 用户指南

## 基本使用

Wallpaper SMTC Music 是 Web 类型 Wallpaper Engine 壁纸。壁纸启动后会优先使用 Wallpaper Engine 官方媒体接口读取当前媒体信息，包括标题、歌手、封面、播放状态和部分播放器提供的时间轴。

如果壁纸设置里开启“无音源是否隐藏”，没有可用媒体时播放器会隐藏，只显示背景。

## Bridge 可选增强

`bridge/WallpaperMusicBridge.exe` 是可选增强程序。不开 Bridge 时，壁纸仍应显示官方媒体接口能提供的信息；开 Bridge 后才启用以下能力：

- 多媒体源列表和媒体源切换。
- 指定媒体源播放/暂停、上一首、下一首。
- 音量加减。
- 更完整、更稳定的播放进度。

Bridge 是托盘程序。重复启动同一路径的 exe 不会创建第二个长期运行实例，而是唤起状态窗口。

托盘菜单：

```text
开机启动
启动服务
停止服务
重启服务
打开状态窗口
退出
```

“服务”指 Bridge 内部 HTTP/SMTC 服务，不是 Windows Service。停止服务会释放 `127.0.0.1:18768` 端口，但托盘程序仍在。

## 壁纸交互

- 点击封面：在播放器和迷你播放器之间临时切换。
- 点击歌名：在样式 A 和样式 B 之间临时切换。
- 临时切换不会写回 Wallpaper Engine 设置；重启电脑或重新加载壁纸后会回到 WE 设置里的默认值。

## 播放器样式

样式 A 是标准播放器，包含封面、歌名、歌手、进度条和控制栏。展开歌词时，播放器整体向左移动，右侧显示歌词。

样式 B 是沉浸歌词样式。它只影响展开后的播放器视觉，不改变媒体源选择、播放控制或歌词数据来源。迷你播放器仍保持自己的控制栏。

## 媒体源规则

媒体源规则只在 Bridge 服务可用时生效。

- 低优先级媒体源：有普通媒体源时不显示，只有没有普通媒体源时才显示。
- 屏蔽媒体源：始终隐藏。

匹配规则是 `sourceId` 子串匹配，大小写不敏感。多个关键词可以用逗号、分号或换行分隔。

示例：

```text
低优先级媒体源：edge, chrome, firefox, brave, opera, browser
屏蔽媒体源：edge
```

## 背景

背景有两条路径：

- 未启用封面背景：使用自定义壁纸。
- 启用封面背景：有媒体封面时优先使用封面背景；无音源时回退自定义壁纸。

自定义壁纸模糊和封面背景模糊是两个独立设置。切歌时如果新封面暂时没有加载出来，壁纸会尽量保留当前背景，避免闪回默认图。

## 常见问题

### 不开 Bridge 能不能控制播放？

通常不能。Wallpaper Engine 官方 Web 媒体接口主要提供媒体信息，不提供稳定的网页播放控制接口。播放控制、音量和媒体源切换依赖 Bridge。

### Bridge 开着但按钮不可用？

先打开 Bridge 状态窗口，确认服务状态是“运行中”，HTTP 地址为 `http://127.0.0.1:18768`。如果服务停止，点击“启动服务”或“重启服务”。

### 切换媒体源看不到某个播放器？

确认该播放器已经向 Windows SMTC 暴露媒体 session。部分播放器只有播放过或启用相关插件后才会出现在 SMTC 中。

### 在线歌词是否联网？

会。开启在线歌词查询时，壁纸会向配置的歌词源发送歌名、歌手等信息。关闭“在线歌词查询”后不主动请求在线歌词源。
